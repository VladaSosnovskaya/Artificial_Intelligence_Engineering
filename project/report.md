## 1. Паспорт проекта

- **Название проекта:** `Классификация эмоций по изображениям лиц`
- **Автор:** `Сосновская Владислава Евгеньевна`
- **Группа:** `БСБО-04-22>`
- **Контакт:** `@vladaS36`
- **Ссылка на репозиторий:** `https://github.com/VladaSosnovskaya/Artificial_Intelligence_Engineering/tree/main`

Проект представляет собой веб-сервис для автоматического распознавания эмоций на изображениях человеческих лиц. На вход подаётся фотография, которая приводится к размеру 48×48 пикселей. На выходе выводится предсказанная эмоция (одна из 7 категорий) с оценкой уверенности модели.    
Что получаем: полноценный пайплайн, демонстрирующий полный цикл разработки: анализ данных, обучение моделей,  сравнение подходов, упаковка в Docker, запуск API с мониторингом

## 6. Архитектура решения и сервис

Опишите, как из модели получился работающий сервис:

1. **Архитектура пайплайна:**
   Пользователь загружает изображение и нажимает на кнопку для классификации
   -> [FastAPI: /predict] изображение передается модели для предобработки предсказания
   -> Модель возвращает JSON ответ с предсказанной эмоцией и уверенностью
   -> результат показывается пользователю  

2. **API и endpoints:**
   - /health - Проверка загрузки модели
   - /predict - Предсказание эмоции (принимает изображение, возвращает эмоцию и уверенность)
   - /metrics - Статистика запросов

3. **Технологический стек:**
   - FastAPI + Uvicorn
   - PyTorch, torchvision, scikit-learn 
   - MLflow для логирования
   - pytest + TestClient для тетов
   - Docker + Docker Compose (CPU/GPU профили)

Запуск докера:
```
docker compose up -d --build
```
   - при этом в .env должен быть указан GPU/CPU
