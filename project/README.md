## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - модули для работы с данными;
  - обучение моделей;
  - запуск сервиса/скриптов.
- `data/` – демонстрационные/учебные данные:
  - **только открытые, синтетические или обезличенные данные;**
  - при необходимости – небольшие выборки для быстрого запуска.
- `configs/` – конфигурационные файлы:
  - `.yaml`, `.json`, `.env.example` и т.п.
- `tests/` – тесты (юнит-тесты, простые проверки).
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.

Если структура проекта отличается от базовой, опишите это здесь.

## 3. Требования и установка

### 3.1. Требования

- Python `= 3.10.12`
- python3.10-venv

### 3.2. Установка окружения
Проект можно запускать как с помощью виртуального окружения, так и с помощью Docker

#### Сценарий запуска с использованием venv
```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение
# Windows:
py -3.10 -m venv .venv
# Linux:
python3.10 -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
# Для CPU:
pip install --no-cache-dir torch==2.6.0 torchvision==0.21.0 --index-url https://download.pytorch.org/whl/cpu
# Для GPU:
pip install --no-cache-dir torch==2.6.0 torchvision==0.21.0 --index-url https://download.pytorch.org/whl/cu124
# Все остальные библиотеки:
pip install -r requirements.txt
# Устанавливить проект в режим разработки
pip install -e .
```

#### Запуск сценария с использование Docker
```bash
docker compose up -d --build
```

## 4. Как запустить проект

Опишите, как именно воспроизвести основные сценарии использования проекта.

### 4.1. Заполнить .env по шаблону configs/.env.example и перенести его в корень проекта (чтобы он лежал на одном уровне с docker-compose)
Для скачивания датасета с kaggle автоматически, получить ключ на "kaggle" -> "settings"
![alt text](images_for_readme/image.png)

### 4.2. Подготовить датасет по инструкции из `data/README.md`

### 4.3. Запуск обучения модели

```bash
# Из venv или внутри Docker
python -m src.train
```
По умолчанию используется конфиг из training_best.yaml. Либо можно указать свои параметры для обучения (любые из указанных):
```bash
# Из venv или внутри Docker
python -m src.train --model_name CNN_model --epochs 1 --lr 0.01 --wd 0.001 --patience 5 --batch_size 64 --augmentation None
``` 
Выбор типа аугментации из списка: base/None, geo, photo, combined

### 4.4. Запуск сервиса (API/веб-интерфейс)

```bash
# Из venv
uvicorn src.service.api:app
```

Либо с помощью докера сервис поднимается автоматически


По умолчаанию сервис поднимается на порту 8000. Эндпоинты:
   - /health - Проверка загрузки модели
   - /predict - Предсказание эмоции (принимает изображение, возвращает эмоцию и уверенность)
   - /metrics - Статистика запросов

Работоспособность можно протестировать в браузере, открыв `localhost:8000`, либо с помощью командной строки:
```bash
curl -s http://localhost:8000/health
# вывод: {"status":"healthy","device":"cpu"}

curl -s http://localhost:8000/metrics
# вывод: {"request_counts":{"/predict":1,"/predict_latency":0.21071934700012207},"avg_latency_ms":210.72}

curl -s -X POST -F "file=@data/train/angry/Training_3908.jpg" http://localhost:8000/predict
```

## 8. Ограничения и дальнейшая работа

Текущие ограничения:
  - Датасет: FER-2013 имеет низкое разрешение (48×48) и заметный дисбаланс классов, что ограничивает потолок точности 
  - Модель: Архитектура обучается с нуля без использования предобученных сетей 
  - Сервис: Обрабатывает только одиночные изображения, без детекции лиц на кадре
  
Направления развития:
  - Данные и модели: Переход на современные датасеты, fine-tuning предобученных архитектур
  - Сервис и инфраструктура: Внедрение Prometheus/Grafana для наблюдаемости, базовая аутентификация и лимиты запросов
  - Функционал: Детекция нескольких лиц на одном изображении, оценка интенсивности эмоции
